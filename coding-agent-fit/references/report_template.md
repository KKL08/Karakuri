# 报告模板

输出简体中文。语气按技术文档：直接、清楚、有证据。

## 文风规则

### 判断段（各维度"判断"字段）

1. **从 Agent 体验写，不从扫描器视角写。** 写"Agent 走到这一步会怎样"，不写"我们检测到 X 缺失"。读者要的是体感，不是扫描结果清单。
2. **说清为什么是这档，不是相邻档。** 不只写"给 4 分"；写"有 X 所以不是 3，缺 Y 所以不是 5"。
3. **区分"未发现"和"不存在"。** probe / fetch / search 没看到 ≠ 服务没提供。没找到时写"本次评测未发现"，不写"缺失"或"没有"。

### 改进建议段（各维度"建议"字段 + "最该先改的 3 件事"）

1. **主语用站点或第三人称，评测方用"我们"。** 不预设读者身份——不用"你"，不用"如果你是 owner"。
2. **格式："我们看到的"→"改进方向"。** 前者陈述事实和对 Agent 接入的实际影响；后者给出站点可以做的事和衡量标准。
3. **面向使用本 skill 做评测的用户。** 报告是评测方写给使用者的交付物。改进建议是基于评测发现提出的优化方向——读者拿去推动站点改进、评估接入风险、比较多家服务，按自己角色决定怎么用。
